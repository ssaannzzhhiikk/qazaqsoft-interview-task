# Quiz Module

Мини-модуль прохождения теста для образовательной платформы. Проект написан на чистых HTML, CSS и JavaScript без сборщика и внешних зависимостей.

## Что умеет

- Загружает вопросы из `data/questions.json`.
- Показывает прогресс в формате `Вопрос X из N`.
- Поддерживает одиночный выбор ответа.
- Позволяет переходить назад и вперед с восстановлением выбранных ответов.
- Считает результат: количество правильных ответов, процент и статус прохождения.
- Автоматически завершает тест, когда таймер доходит до нуля.
- Сохраняет прогресс в `localStorage` и восстанавливает его после перезагрузки.
- Показывает режим просмотра ответов с подсветкой правильных, ошибочных и пропущенных ответов.

## Как запустить

Из-за загрузки `data/questions.json` через `fetch` проект нужно открыть через локальный сервер.

```powershell
cd c:\Users\user\Desktop\qazaqsoft-interview-task
python -m http.server 8000
```

После запуска откройте:

```text
http://localhost:8000
```

Если порт занят, можно указать другой:

```powershell
python -m http.server 8080
```

## Архитектура

- `Question` - модель одного вопроса.
- `QuizEngine` - управление состоянием теста: текущий вопрос, ответы, таймер, завершение и расчет результата.
- `StorageService` - чтение, запись и очистка сохраненного прогресса.
- DOM-функции в `app.js` отвечают только за загрузку данных, обработку событий и рендер интерфейса.

## Формат данных

```json
{
  "title": "Основы JavaScript",
  "timeLimitSec": 300,
  "passThreshold": 0.7,
  "questions": [
    {
      "id": "q1",
      "text": "Какой тип данных у значения null?",
      "options": ["object", "null", "undefined", "number"],
      "correctIndex": 0
    }
  ]
}
```

`timeLimitSec` и `passThreshold` можно не указывать. По умолчанию используются `300` секунд и порог `0.7`.

## Валидация данных

При загрузке проверяются:

- наличие заголовка;
- непустой массив вопросов;
- уникальные `id`;
- текст вопроса;
- минимум два варианта ответа;
- корректный `correctIndex`.

Если данные некорректны, пользователь увидит понятное сообщение об ошибке.

## Сохранение прогресса

В `localStorage` сохраняются:

- текущий вопрос;
- выбранные ответы;
- оставшееся время;
- статус завершения;
- версия и сигнатура теста.

Если файл вопросов изменился, старое сохранение не восстанавливается, чтобы не смешивать ответы от другой версии теста.

## Ограничения

- Нет unit-тестов.
- Нет перемешивания вопросов и вариантов.
- Нет подробной аналитики по времени на каждый вопрос.
- Интерфейс минимальный и рассчитан на тестовое задание, а не на полноценный дизайн-системный продукт.
